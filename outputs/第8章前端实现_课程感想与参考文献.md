# 8 前端实现

## 8.1 实现环境说明

Web 酒店管理系统需要将界面交互、业务处理和数据存储进行分层，以便各功能模块能够独立开发和维护[2-5]。本系统采用 B/S 架构和前后端分离模式实现：前端负责页面展示、表单校验及用户操作，后端负责身份验证、业务处理和数据访问，前后端通过 HTTP 接口传递 JSON 数据。系统的主要开发与运行环境如表 8-1 所示。

**表 8-1 系统实现环境**

| 类别 | 技术或工具 | 版本及用途 |
|:---:|:---:|:---|
| 操作系统 | Windows | 项目开发、数据库部署及系统运行环境 |
| 前端框架 | Vue.js | 2.6.10，用于构建组件化单页面应用[7] |
| 前端组件库 | Element UI | 2.13.0，用于实现表单、表格、分页和消息提示等界面组件[10] |
| 网络请求 | Axios | 0.19.2，用于前后端异步数据交互 |
| 前端构建工具 | Vue CLI | 4.1.0，前端开发服务端口为 8081 |
| 后端语言 | Java | JDK 1.8 |
| 后端框架 | Spring Boot | 2.2.2.RELEASE，用于构建 Web 服务和接口层[6] |
| 持久层框架 | MyBatis、MyBatis-Plus | MyBatis Spring Boot Starter 2.1.1、MyBatis-Plus 2.3，用于数据映射、条件查询和分页处理[9] |
| 数据库 | MySQL | 使用 `hotel_management` 数据库保存系统业务数据[8] |
| 项目管理工具 | Maven | 管理后端依赖并完成项目构建 |
| Web 服务器 | Spring Boot 内嵌 Tomcat | 后端服务端口为 8080，访问上下文为 `/springbootb1g8z` |

前端通过 Axios 创建统一请求实例，并将接口基础地址设置为 `/springbootb1g8z`。在开发环境中，Vue 开发服务器把该路径下的请求代理至 `http://localhost:8080/springbootb1g8z/`，从而避免前后端端口不同造成的跨域问题。请求拦截器会把本地存储的 Token 写入请求头，响应拦截器则统一处理身份失效情况。

```javascript
const http = axios.create({
  timeout: 1000 * 86400,
  withCredentials: true,
  baseURL: '/springbootb1g8z',
  headers: {
    'Content-Type': 'application/json; charset=utf-8'
  }
})

http.interceptors.request.use(config => {
  config.headers.Token = storage.get('Token')
  return config
})

http.interceptors.response.use(response => {
  if (response.data && response.data.code === 401) {
    router.push({ name: 'login' })
  }
  return response
})
```

## 8.2 登录界面

系统登录界面由角色选择、账号输入、密码输入和登录按钮组成。后台登录角色包括管理员和员工，用户选择角色后，前端会根据角色对应的数据表名称动态确定登录接口。例如，管理员请求 `admin_user/login`，员工请求 `employee/login`。这种实现方式复用了同一套登录页面，同时使不同身份能够进入与其权限相匹配的后台功能。

> **此处插入登录界面截图**
>
> 图 8-1 系统登录界面

用户单击“登录”按钮后，前端首先检查账号和密码是否为空；校验通过后，将表单数据按 `application/x-www-form-urlencoded` 格式发送至后端。登录成功时，系统保存后端返回的 Token、当前角色、会话表名和登录账号，并跳转到后台首页；登录失败时，通过消息组件显示后端返回的错误信息。前端登录的关键代码如下。

```javascript
login() {
  if (!this.form.username || !this.form.password) {
    this.$message.error('请输入账号和密码')
    return
  }

  const formData = new URLSearchParams()
  formData.append('username', this.form.username)
  formData.append('password', this.form.password)

  this.$http({
    url: `${this.form.tableName}/login`,
    method: 'post',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded'
    },
    data: formData
  }).then(({ data }) => {
    if (data && data.code === 0) {
      const role = this.roles.find(
        item => item.tableName === this.form.tableName
      )
      this.$storage.set('Token', data.auth_token)
      this.$storage.set('role', role ? role.roleName : '')
      this.$storage.set('sessionTable', this.form.tableName)
      this.$storage.set('adminName', this.form.username)
      this.$router.replace('/index')
    } else {
      this.$message.error(data.msg || '登录失败')
    }
  })
}
```

后端接收账号和密码后，根据账号查询对应用户记录。当前课程设计版本采用账号、密码比对完成身份校验；校验成功后，由 `TokenService` 生成访问令牌并返回前端。此后访问需要授权的接口时，前端会在请求头中携带该令牌，后端据此识别用户身份和角色。管理员登录接口的核心代码如下。

```java
@IgnoreAuth
@PostMapping(value = "/login")
public R login(String username, String password,
               String captcha, HttpServletRequest request) {
    UserEntity user = userService.selectOne(
        new EntityWrapper<UserEntity>().eq("username", username)
    );

    if (user == null || !user.getPassword().equals(password)) {
        return R.error("账号或密码不正确");
    }

    String authToken = auth_tokenService.generateToken(
        user.getId(),
        username,
        "admin_user",
        user.getRole()
    );
    return R.ok().put("auth_token", authToken);
}
```

上述登录流程实现了“角色选择—身份校验—令牌生成—页面跳转”的完整过程。页面按钮和菜单还会结合当前角色的权限配置进行显示，避免普通员工进入无权操作的管理模块。

## 8.3 数据管理界面

本节选择客房资源管理中的“客房目录”模块说明数据管理界面的实现。该模块用于维护客房名称、客房类型、客房图片、客房价格、客房数量、所属酒店、酒店地址、客房设施和客房介绍等信息，同时展示评价数量和平均评分等扩展数据。管理员可在列表页按客房名称、客房类型和酒店名称组合查询，并根据权限执行查看、新增、修改、删除及批量删除操作。

> **此处插入客房目录数据管理界面截图**
>
> 图 8-2 客房目录数据管理界面

客房目录列表使用 Element UI 的表格和分页组件展示查询结果。前端根据当前页码、每页记录数和查询条件构造请求参数，其中客房名称和酒店名称采用模糊查询，客房类型采用精确匹配。后端返回数据后，前端分别更新列表内容和总记录数。分页查询的关键代码如下。

```javascript
getDataList() {
  this.dataListLoading = true
  const params = {
    page: this.pageIndex,
    limit: this.pageSize,
    sort: 'id'
  }

  if (this.searchForm.kefangmingcheng) {
    params.kefangmingcheng =
      `%${this.searchForm.kefangmingcheng}%`
  }
  if (this.searchForm.room_category) {
    params.room_category = this.searchForm.room_category
  }
  if (this.searchForm.jiudianmingcheng) {
    params.jiudianmingcheng =
      `%${this.searchForm.jiudianmingcheng}%`
  }

  this.$http({
    url: 'room_catalog/page',
    method: 'get',
    params
  }).then(({ data }) => {
    if (data && data.code === 0) {
      this.dataList = data.data.list
      this.totalPage = data.data.total
    } else {
      this.dataList = []
      this.totalPage = 0
    }
    this.dataListLoading = false
  })
}
```

列表中的操作按钮通过 `isAuth` 判断当前角色是否具有相应权限。新增操作不携带记录编号，修改和查看操作则把所选记录的主键传递给编辑组件；查看状态下表单字段为只读。相应的页面调用代码如下。

```javascript
addOrUpdateHandler(id, type) {

  if (type !== 'info') {
    type = 'else'

    this.$refs.addOrUpdate.init(id, type)

```

新增和修改共用同一表单组件。页面使用 `v-model` 完成表单控件与 `ruleForm` 对象的双向绑定，并通过表单校验规则检查必填项及数字字段。图片通过文件上传组件提交，客房介绍使用富文本编辑器录入。提交时，前端根据数据对象中是否存在 `id` 自动选择 `save` 或 `update` 接口，因此无需分别维护两套表单。

```javascript
this.$refs.ruleForm.validate(valid => {
  if (!valid) {
    return
  }

  const action = this.ruleForm.id ? 'update' : 'save'
  this.$http({
    url: `room_catalog/${action}`,
    method: 'post',
    data: this.ruleForm
  }).then(({ data }) => {
    if (data && data.code === 0) {
      this.$message.success('操作成功')
      this.parent.showFlag = true
      this.parent.addOrUpdateFlag = false
      this.parent.search()
    } else {
      this.$message.error(data.msg)
    }
  })
})
```

删除功能既支持删除单条记录，也支持在表格中勾选多条记录后进行批量删除。前端先整理待删除记录的主键数组，并在用户确认后调用删除接口。删除完成后重新执行列表查询，使页面内容与数据库保持一致。

```javascript
deleteHandler(id) {
  const ids = id
    ? [Number(id)]
    : this.dataListSelections.map(item => Number(item.id))

  this.$confirm(
    `确定进行${id ? '删除' : '批量删除'}操作？`,
    '提示',
    { type: 'warning' }
  ).then(() => {
    this.$http({
      url: 'room_catalog/delete',
      method: 'post',
      data: ids
    }).then(({ data }) => {
      if (data && data.code === 0) {
        this.$message.success('操作成功')
        this.search()
      } else {
        this.$message.error(data.msg)
      }
    })
  })
}
```

后端控制器为客房目录提供分页查询、新增、修改和删除接口。分页接口使用 MyBatis-Plus 的条件包装器处理查询参数；新增和修改数据前调用 `syncResourceRelation`，根据酒店名称和客房类型查询关联记录，并自动补充 `hotel_profile_id`、`room_category_id` 和酒店地址，从而降低名称字段与外键字段不一致的风险。主要接口代码如下。

```java
@RequestMapping("/page")
public R page(@RequestParam Map<String, Object> params,
              JiudiankefangEntity roomCatalog) {
    EntityWrapper<JiudiankefangEntity> wrapper =
        new EntityWrapper<JiudiankefangEntity>();
    PageUtils page = roomCatalogService.queryPage(
        params,
        MPUtil.sort(
            MPUtil.between(
                MPUtil.likeOrEq(wrapper, roomCatalog),
                params
            ),
            params
        )
    );
    return R.ok().put("data", page);
}

@RequestMapping("/save")
public R save(@RequestBody JiudiankefangEntity roomCatalog) {
    syncResourceRelation(roomCatalog);
    roomCatalogService.insert(roomCatalog);
    return R.ok();
}

@RequestMapping("/update")
public R update(@RequestBody JiudiankefangEntity roomCatalog) {
    syncResourceRelation(roomCatalog);
    roomCatalogService.updateById(roomCatalog);
    return R.ok();
}

@RequestMapping("/delete")
public R delete(@RequestBody Long[] ids) {
    roomCatalogService.deleteBatchIds(Arrays.asList(ids));
    return R.ok();
}
```

关联数据同步代码如下。

```java
private void syncResourceRelation(
        JiudiankefangEntity roomCatalog) {
    if (roomCatalog == null) {
        return;
    }

    if (StringUtils.isNotBlank(
            roomCatalog.getJiudianmingcheng())) {
        JiudianjianjieEntity hotel =
            hotelProfileService.selectOne(
                new EntityWrapper<JiudianjianjieEntity>()
                    .eq("jiudianmingcheng",
                        roomCatalog.getJiudianmingcheng())
            );
        if (hotel != null) {
            roomCatalog.setHotelProfileId(hotel.getId());
            if (StringUtils.isBlank(
                    roomCatalog.getJiudiandizhi())) {
                roomCatalog.setJiudiandizhi(
                    hotel.getJiudiandizhi());
            }
        }
    }

    if (StringUtils.isNotBlank(
            roomCatalog.getRoom_category())) {
        KefangleixingEntity category =
            roomCategoryService.selectOne(
                new EntityWrapper<KefangleixingEntity>()
                    .eq("room_category",
                        roomCatalog.getRoom_category())
            );
        if (category != null) {
            roomCatalog.setRoomCategoryId(category.getId());
        }
    }
}
```

通过上述实现，客房目录模块形成了从条件查询、分页展示到新增、修改和批量删除的完整数据管理流程。前端负责交互和输入校验，后端负责业务规则、关联字段同步和持久化操作，使客房基础资料能够为预订、入住、房态、价格策略和评价等后续业务提供统一的数据来源。
